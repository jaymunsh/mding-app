# mding PWA Design

## Goal

Rebuild mding as a local-first installable PWA for iOS, iPadOS, and macOS. The app should feel close to a lightweight native editor while avoiding Apple Developer Program and TestFlight constraints.

## Product Shape

mding owns an internal Markdown workspace stored in browser storage. The user manages folders and Markdown files inside the app, opens files from a tree, previews rendered Markdown, switches to source editing, and imports/exports content for backup or iCloud Drive transfer.

## Platform Behavior

- iPhone: single-pane flow. The file tree opens first, then selecting a file moves to the document pane. A compact toolbar provides back, edit, save, and file actions.
- iPad/macOS/narrow desktop browser: split view with file tree on the left and document pane on the right.
- macOS: keyboard and pointer use must feel natural, with visible hover/focus states, larger split view, and import/export from normal browser file pickers.
- PWA install: app is hosted as static files, installed from Safari's Add to Home Screen on iOS, and can run offline after first load.

## Storage

The workspace uses IndexedDB as the durable source of truth:

- A metadata store tracks file and folder nodes.
- A document store tracks Markdown text by file id.
- The app requests persistent storage when available.
- The app can export the entire workspace as a JSON backup file for now. Zip export can be layered later without changing the workspace model.

IDs are stable strings generated by `crypto.randomUUID()`. Parent-child relationships model folders and files without relying on OS paths.

## Core Features

- Create Markdown files and folders.
- Rename files and folders.
- Delete files and folders.
- Move files and folders within the internal workspace.
- Select file from the tree.
- Preview Markdown.
- Edit Markdown source.
- Save edits locally.
- Import `.md` files into the selected folder.
- Export selected file as `.md`.
- Export entire workspace as `.json`.
- Import workspace `.json`.
- Work offline after install.

## Architecture

- `src/domain`: pure workspace types and tree-building logic.
- `src/storage`: IndexedDB adapter, persistence request, import/export helpers.
- `src/app`: React state orchestration and reducer.
- `src/components`: UI components built from `DESIGN.md` primitives.
- `src/styles`: app-wide CSS tokens and component styling.

The app avoids heavy editor packages in the MVP. A textarea handles Markdown source editing, and a small Markdown renderer dependency handles preview.

## Error Handling

Expected failures, such as duplicate names or missing selections, return typed `Result` values. Unexpected storage failures are converted into user-visible app errors at the storage boundary.

## Verification

- Type check with `tsc --noEmit`.
- Lint/format with Biome.
- Unit tests with Vitest for tree building, reducer actions, and storage import/export helpers.
- Production build with Vite.
- Browser QA through local preview at mobile, tablet, and desktop widths.

## Out of Scope

- Real-time cloud sync.
- Direct iCloud Drive folder mounting.
- Rich editor libraries such as Monaco or CodeMirror.
- Collaborative editing.
- Native app wrappers.
